# Orders & Products Management System

Високопродуктивний full-stack веб-додаток, розроблений для відстеження складських запасів у реальному часі, керування приходами (замовленнями) та моніторингу активних сесій. Побудований на базі Next.js 16 (App Router), Express, Prisma, PostgreSQL та Socket.io.

Вся система повністю контейнеризована і готова до розгортання через Docker Compose з автоматичним накочуванням міграцій баз даних та заповненням тестовими даними (seed).

## 🚀 Технологічний стек

### Frontend

- **Framework**: Next.js 16 (App Router, оптимізований під Production Build)
- **Language**: TypeScript
- **State Management**: Redux Toolkit (RTK)
- **Styling & UI**: Bootstrap 5 / React-Bootstrap
- **Real-Time Integration**: Socket.io-client
- **Localization**: Власна i18n-система на базі React Context (підтримка UA, EN, RU)
- **Animations**: Нативні CSS Keyframes з оптимізацією під GPU (`will-change: transform`)

### Backend & Database

- **Runtime**: Node.js з Express
- **ORM**: Prisma ORM (PostgreSQL Provider)
- **Database**: PostgreSQL (Контейнеризована, з персистентними волюмами даних)
- **Real-Time Engine**: Socket.io (Подійно-орієнтований трекер підключених вкладок)

## 🏗️ Ключові можливості системи

1. **Сторінка приходів (`/orders`)**:
   - Інтерактивні списки з динамічним розрахунком загальної вартості замовлень одночасно в **USD** та **UAH**.
   - Раздвижна панель детального перегляду (Detail View), яка плавно відкривається при кліку, відображаючи пов'язані з цим приходом продукти.
   - Безпечне каскадне видалення приходів та ізольоване видалення окремих продуктів через модальні вікна підтвердження.
2. **Сторінка продуктів (`/products`)**:
   - Пласка таблична структура з відображенням серійних номерів, специфікацій, термінів гарантії та назви головного приходу.
   - Динамічна фільтрація за типом продукту, керована реактивним станом.
3. **TopMenu та синхронізація в реальному часі**:
   - Фіксована верхня навігаційна панель (TopMenu), що залишається доступною при скролінгу великих таблиць.
   - `LiveClock`: Мультимовний годинник реального часу, який оновлюється щосекунди з урахуванням обраної мови сайту.
   - `ActiveSessions`: WebSocket-лічильник відкритих вкладок, який миттєво транслює та анімує зміни, коли користувач відкриває або закриває кілька вкладок/браузерів одночасно.
4. **Безпека та роутинг**:
   - Автоматичний серверний редірект із кореня `/` на `/orders`, що виключає відображення порожніх екранів-заглушок.
   - Суворий стан заблокованих роутів (`Disabled States`) для функцій, що знаходяться в розробці (Групи, Користувачі, Налаштування), з використанням UX-friendly курсорів `not-allowed` та чітких міток для запобігання помилок 404.

## 📦 Архітектура та структура репозиторію

Проєкт структурований як **монорепозиторій**, що забезпечує чітку ізоляцію між клієнтською логікою та моделями бази даних:

```text
orders-monorepo/
├── backend/            # Express, Node.js, Prisma REST & WS Контролери
│   ├── prisma/         # Схеми бази даних та скрипти заповнення (seed)
│   └── src/            # Ізольовані роути, сокети та конфігурація сервера
├── frontend/           # Клієнтський додаток Next.js 16 (TypeScript + RTK)
│   └── src/app/        # Сторінки App Router, контексти та спільні компоненти
└── docker-compose.yml  # Головний оркестратор для розгортання всього проекту
```

## ⚡ Швидкий запуск (Production запуск через Docker)

Найпростіший спосіб клонувати, підняти та протестувати всю екосистему (Базу даних, Бекенд та Фронтенд) — використовувати **Docker Compose**.

### 1️⃣ Клонування репозиторію

Переконайтеся, що на вашому комп'ютері запущено Docker двигун (Docker Desktop / Docker Daemon).

```bash
git clone <url-вашого-репозиторію>
cd orders-monorepo

```

### 2️⃣ Запуск системи

Виконайте наступну єдину команду в кореневій папці проекту. Вона завантажить базові образи, збере ізольовані шари додатків, автоматично накотить міграції Prisma, запустить сид тестових даних та підніме сервери:

```bash
docker-compose up --build

```

### 3️⃣ Доступ до додатку

Після успішного завершення скриптів збірки та сидування, відкрийте браузер:

- **Frontend інтерфейс**: `http://localhost:3000` _(Автоматично перенаправить на `/orders`)_
- **Backend API сервер**: `http://localhost:3001`
- **Доступ до БД**: Доступний локально на порту `5433` _(Налаштовано саме на 5433, щоб повністю ізолювати базу проекту від будь-яких нативних версій PostgreSQL, які можуть працювати у вас на стандартному 5432 порту)_.

## 🛠️ Ручне налаштування для локальної розробки

Якщо ви бажаєте запустити процеси додатків вручну без використання Docker-контейнерів:

### Вимоги

Переконайтеся, що у вас встановлено **Node.js v20+** та запущено екземпляр **PostgreSQL**.

### 🔌 1. Налаштування бекенду

1. Перейдіть до папки бекенду та інсталюйте залежності:

```bash
cd backend
npm install

```

2. Створіть файл `.env` всередині папки `backend/` та вкажіть змінні доступу до бази:

```env
PORT=3001
DATABASE_URL="postgresql://postgres:secret_password_2026@localhost:5433/orders_db?schema=public"

```

3. Ініціалізуйте структуру бази даних та заповніть її тестовими наборами:

```bash
npx prisma migrate dev --name init
npx prisma db seed

```

4. Запустіть сервер розробки:

```bash
npm run dev

```

### 💻 2. Налаштування фронтенду

1. Перейдіть до робочого простору фронтенду та інсталюйте пакети:

```bash
cd ../frontend
npm install

```

2. Налаштуйте адресу підключення до бекенду у файлі `frontend/.env.local`:

```env
NEXT_PUBLIC_API_URL="http://localhost:3001/api"

```

3. Запустіть клієнтський двигун Next.js:

```bash
npm run dev

```
